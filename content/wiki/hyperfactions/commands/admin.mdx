# Admin Commands

These commands are for server administrators to manage factions, zones, backups, and the plugin itself. Admin commands require permissions from the `hyperfactions.admin.*` category.

## Admin Menu

<CommandBlock
  commands={[
    {
      command: "/f admin",
      description: "Open the admin management GUI",
      permission: "hyperfactions.admin.use"
    }
  ]}
/>

The admin menu provides GUI access to all administrative functions including faction management, zone creation, and system settings.

## Configuration Commands

<CommandBlock
  commands={[
    {
      command: "/f admin reload",
      description: "Reload the configuration file",
      permission: "hyperfactions.admin.reload"
    },
    {
      command: "/f admin debug",
      description: "Toggle debug mode for troubleshooting",
      permission: "hyperfactions.admin.debug"
    }
  ]}
/>

**Reload applies:**
- `config.json` settings
- Message configuration
- Power settings
- All other configuration changes

<InfoBox variant="info">
Some changes may require a server restart. The reload command handles most configuration changes.
</InfoBox>

**Debug mode:**
- Shows detailed logging for troubleshooting
- Outputs to console and/or chat
- Categories can be configured in `config.json`

## Zone Management

The zone system allows admins to create named zones that span multiple chunks with customizable flags.

### Creating Zones

<CommandBlock
  commands={[
    {
      command: "/f admin zone create <name> <safe|war>",
      description: "Create a new named zone (SafeZone or WarZone)",
      permission: "hyperfactions.admin.zones"
    }
  ]}
/>

**Zone Types:**
- `safe` - SafeZone (no PvP, no building, protected area)
- `war` - WarZone (PvP enabled, no power loss by default)

### Claiming Chunks for Zones

<CommandBlock
  commands={[
    {
      command: "/f admin zone claim <name>",
      description: "Add the current chunk to an existing zone",
      permission: "hyperfactions.admin.zones"
    },
    {
      command: "/f admin zone radius <name> <radius> [circle]",
      description: "Claim a radius of chunks for a zone",
      permission: "hyperfactions.admin.zones"
    }
  ]}
/>

**Radius claiming:**
- `radius` - Number of chunks from current position
- Add `circle` to claim in a circular pattern instead of square

**Example:** `/f admin zone radius spawn 5 circle` claims a circular area with 5-chunk radius.

### Managing Zones

<CommandBlock
  commands={[
    {
      command: "/f admin zone unclaim",
      description: "Remove the current chunk from its zone",
      permission: "hyperfactions.admin.zones"
    },
    {
      command: "/f admin zone remove <name>",
      description: "Delete a zone entirely (unclaims all chunks)",
      permission: "hyperfactions.admin.zones"
    },
    {
      command: "/f admin zone list [safe|war]",
      description: "List all zones, optionally filtered by type",
      permission: "hyperfactions.admin.zones"
    }
  ]}
/>

### Zone Flags

Each zone type has configurable flags. Use the admin GUI to modify zone flags:

| Flag | SafeZone Default | WarZone Default | Description |
|------|------------------|-----------------|-------------|
| `pvp_enabled` | false | true | Allow PvP |
| `friendly_fire` | false | true | Allow damage between allies |
| `build_allowed` | false | false | Allow building |
| `container_access` | false | false | Allow container access |
| `interact_allowed` | true | true | Allow interactions |
| `item_drop` | true | true | Allow item dropping |
| `item_pickup` | true | true | Allow item pickup |
| `mob_spawning` | false | true | Allow mob spawning |
| `mob_damage` | false | true | Allow mob damage |
| `hunger_loss` | false | true | Allow hunger drain |
| `fall_damage` | false | true | Allow fall damage |

## Bypass Mode

<CommandBlock
  commands={[
    {
      command: "/f admin bypass",
      description: "Toggle admin bypass mode",
      permission: "hyperfactions.admin.use"
    }
  ]}
/>

**Bypass Mode Enables:**
- Build in any claimed territory
- Open containers anywhere
- Interact with mechanisms
- Useful for investigating or fixing issues

**Toggle:**
- Run once to enable
- Run again to disable

<InfoBox variant="tip">
Remember to disable bypass mode when done to avoid accidentally breaking player builds.
</InfoBox>

## Faction Management

<CommandBlock
  commands={[
    {
      command: "/f admin disband <faction>",
      description: "Force disband any faction",
      permission: "hyperfactions.admin.disband"
    },
    {
      command: "/f admin modify <faction> <setting> <value>",
      description: "Modify any faction's settings",
      permission: "hyperfactions.admin.modify"
    }
  ]}
/>

**Modifiable settings:**
- `name` - Change faction name
- `description` - Change description
- `color` - Change faction color
- `power` - Set faction power
- `leader` - Change faction leader
- `open` - Set open/closed status

<InfoBox variant="warning">
Admin faction modifications bypass role checks. Use with caution as they directly affect player factions.
</InfoBox>

## Backup System

HyperFactions includes a built-in backup system with GFS (Grandfather-Father-Son) rotation.

<CommandBlock
  commands={[
    {
      command: "/f admin backup create [name]",
      description: "Create a manual backup with optional custom name",
      permission: "hyperfactions.admin.backup"
    },
    {
      command: "/f admin backup list",
      description: "List all available backups",
      permission: "hyperfactions.admin.backup"
    },
    {
      command: "/f admin backup restore <name>",
      description: "Restore from a specific backup",
      permission: "hyperfactions.admin.backup"
    },
    {
      command: "/f admin backup delete <name>",
      description: "Delete a specific backup",
      permission: "hyperfactions.admin.backup"
    }
  ]}
/>

**Backup retention (configurable):**
- Hourly backups: 24 (1 day)
- Daily backups: 7 (1 week)
- Weekly backups: 4 (1 month)

<InfoBox variant="danger" title="Restore Warning">
Restoring a backup overwrites current data. Consider creating a backup before restoring.
</InfoBox>

## Update System

<CommandBlock
  commands={[
    {
      command: "/f admin update",
      description: "Check for and download updates",
      permission: "hyperfactions.admin.update"
    }
  ]}
/>

**Update process:**
1. Checks GitHub releases for new versions
2. Downloads the new JAR file
3. Prompts for server restart to apply

<InfoBox variant="info">
Updates are downloaded but not applied until server restart. This prevents data loss during active sessions.
</InfoBox>

## Example Workflows

### Setting Up Server Spawn

1. **Create the SafeZone:**
   ```
   /f admin zone create spawn safe
   ```

2. **Claim the spawn area:**
   ```
   /f admin zone radius spawn 3 circle
   ```
   This claims a 3-chunk radius circle around your position.

3. **Verify:**
   ```
   /f admin zone list safe
   ```

### Creating a PvP Arena

1. **Build the arena structure first**

2. **Create the WarZone:**
   ```
   /f admin zone create arena war
   ```

3. **Stand in the arena and claim:**
   ```
   /f admin zone claim arena
   ```
   Repeat for each chunk, or use radius:
   ```
   /f admin zone radius arena 2
   ```

4. **Configure no power loss in `config.json`:**
   ```json
   "power": {
     "warzonePowerLoss": false
   }
   ```

5. **Reload:**
   ```
   /f admin reload
   ```

### Investigating Player Issues

1. **Enable bypass:**
   ```
   /f admin bypass
   ```

2. **Enable debug mode if needed:**
   ```
   /f admin debug
   ```

3. **Investigate the area**

4. **Disable bypass and debug when done:**
   ```
   /f admin bypass
   /f admin debug
   ```

### Creating a Manual Backup Before Changes

1. **Create named backup:**
   ```
   /f admin backup create pre-update
   ```

2. **Make your changes**

3. **If something goes wrong:**
   ```
   /f admin backup restore pre-update
   ```

### Removing Inactive Faction

1. **Check faction info first:**
   ```
   /f info InactiveFaction
   ```

2. **Force disband:**
   ```
   /f admin disband InactiveFaction
   ```

## Safety Guidelines

### Do
- Keep bypass mode disabled when not needed
- Document zone changes for other admins
- Create backups before major changes
- Test configuration changes on a staging server first
- Use debug mode to troubleshoot issues

### Don't
- Leave bypass enabled accidentally
- Force disband without investigating
- Restore backups without creating a current backup first
- Modify factions without documenting the reason
