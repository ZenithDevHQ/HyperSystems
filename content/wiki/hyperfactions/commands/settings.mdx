# Settings Commands

These commands allow Leaders and Officers to configure faction settings like name, description, color, and membership policy.

## Renaming Your Faction

<CommandBlock
  commands={[
    {
      command: "/f rename <name>",
      description: "Change your faction's name",
      permission: "hyperfactions.faction.rename"
    }
  ]}
/>

**Requirements:**
- Must be the Leader
- New name must follow naming rules (3-24 chars, alphanumeric)
- Name must not be taken by another faction

<InfoBox variant="warning">
Renaming is visible to everyone. Previous name references may become outdated.
</InfoBox>

## Setting Description

<CommandBlock
  commands={[
    {
      command: "/f desc <text>",
      description: "Set your faction's description",
      permission: "hyperfactions.faction.description"
    }
  ]}
/>

**Requirements:**
- Officer or Leader role
- Description appears in `/f info`

**Example:**
```
/f desc Warriors of the Northern Realm. Recruiting active players!
```

## Setting Faction Tag

<CommandBlock
  commands={[
    {
      command: "/f tag <tag>",
      description: "Set your faction's chat tag",
      permission: "hyperfactions.faction.tag"
    }
  ]}
/>

**Requirements:**
- Must be the Leader
- Tag is shown in chat next to player names

**Example:**
```
/f tag WoN
```

## Setting Faction Color

<CommandBlock
  commands={[
    {
      command: "/f color <hex>",
      description: "Set your faction's display color",
      permission: "hyperfactions.faction.color"
    }
  ]}
/>

**Format:** Use hex color codes with or without `#`

**Examples:**
```
/f color #FF5500
/f color FF5500
/f color #00FF00
```

**Color appears in:**
- Territory map
- Faction chat
- GUI elements
- Faction info display

<InfoBox variant="tip" title="Color Codes">
Use a color picker tool to find hex codes. Common colors:
- Red: `#FF0000`
- Blue: `#0000FF`
- Green: `#00FF00`
- Yellow: `#FFFF00`
- Purple: `#800080`
- Orange: `#FF8000`
</InfoBox>

## Opening Membership

<CommandBlock
  commands={[
    {
      command: "/f open",
      description: "Allow anyone to join without invitation",
      permission: "hyperfactions.faction.open"
    }
  ]}
/>

**Requirements:**
- Must be the Leader
- Anyone can join with `/f join FactionName`

<InfoBox variant="info">
Open factions are good for recruiting but provide less control over membership.
</InfoBox>

## Closing Membership

<CommandBlock
  commands={[
    {
      command: "/f close",
      description: "Require invitations to join",
      permission: "hyperfactions.faction.close"
    }
  ]}
/>

**Requirements:**
- Must be the Leader
- Players must be invited to join

This is the default state for new factions.

## Territory Permissions

<CommandBlock
  commands={[
    {
      command: "/f permissions",
      description: "Open the territory permissions GUI",
      permission: "hyperfactions.faction.permissions"
    }
  ]}
/>

**Available flags:**
- `pvp_enabled` - Allow PvP in territory
- `friendly_fire` - Allow faction members to damage each other
- `build_allowed` - Allow building
- `container_access` - Allow container access
- `interact_allowed` - Allow interactions
- `item_drop` / `item_pickup` - Allow item drop/pickup
- `mob_spawning` / `mob_damage` - Mob settings
- `hunger_loss` / `fall_damage` - Environment effects

See [Factions](/plugins/hyperfactions/wiki/concepts/factions) for details on territory permissions.

## Disbanding

<CommandBlock
  commands={[
    {
      command: "/f disband",
      description: "Dissolve your faction permanently",
      permission: "hyperfactions.faction.disband"
    }
  ]}
/>

**Requirements:**
- Must be the Leader
- Requires confirmation

<InfoBox variant="danger" title="Permanent">
Disbanding cannot be undone. All claims are released and all members are removed.
</InfoBox>

## Quick Reference

| Command | Description | Role Required |
|---------|-------------|---------------|
| `/f rename <name>` | Change faction name | Leader |
| `/f desc <text>` | Set description | Officer+ |
| `/f tag <tag>` | Set chat tag | Leader |
| `/f color <hex>` | Set faction color | Officer+ |
| `/f open` | Allow free joining | Leader |
| `/f close` | Require invitations | Leader |
| `/f permissions` | Configure territory flags | Leader |
| `/f disband` | Dissolve faction | Leader |

## Examples

### Complete Faction Setup

1. **Set a description:**
   ```
   /f desc Elite PvP faction. Must have 100+ hours.
   ```

2. **Choose a color:**
   ```
   /f color #FF0000
   ```

3. **Set a tag:**
   ```
   /f tag ELITE
   ```

4. **Keep it invite-only:**
   ```
   /f close
   ```

### Rebranding

1. **New name:**
   ```
   /f rename Phoenix
   ```

2. **Updated description:**
   ```
   /f desc Risen from the ashes. Stronger than ever.
   ```

3. **New color:**
   ```
   /f color #FF8800
   ```

### Recruiting Phase

```
/f desc Looking for active players! Beginners welcome.
/f open
```

Now anyone can join. Close it after recruiting:

```
/f close
```

## Best Practices

### Choosing a Good Name
- **Memorable** - Easy to type and remember
- **Appropriate** - Follows server rules
- **Unique** - Stands out from others

### Writing a Good Description
- **Informative** - What is your faction about?
- **Recruiting info** - Include requirements if applicable
- **Contact info** - How to reach leadership

### Color Selection
- **Visible** - Avoid colors too close to UI elements
- **Unique** - Try to differ from nearby factions
- **Thematic** - Match your faction's identity
