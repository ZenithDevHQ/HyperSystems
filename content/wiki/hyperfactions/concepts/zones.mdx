# Zones

HyperFactions provides two special zone types for server administration: **SafeZones** and **WarZones**. These are admin-managed areas with special rules that can span multiple chunks.

## Zone System Overview

As of v0.2.0, zones are named entities that can contain multiple chunks. This allows for more flexible zone management compared to single-chunk zones.

**Key features:**
- Named zones (e.g., "spawn", "arena", "market")
- Multi-chunk support
- 11 configurable flags per zone
- GUI-based management
- Radius claiming

## SafeZones

SafeZones are protected areas where PvP is disabled and players are generally safe.

### SafeZone Default Properties

| Flag | Default | Description |
|------|---------|-------------|
| `pvp_enabled` | false | No PvP |
| `friendly_fire` | false | No friendly damage |
| `build_allowed` | false | No building |
| `container_access` | false | No container access |
| `interact_allowed` | true | Can use doors, buttons |
| `item_drop` | true | Can drop items |
| `item_pickup` | true | Can pick up items |
| `mob_spawning` | false | No mob spawning |
| `mob_damage` | false | No mob damage |
| `hunger_loss` | false | No hunger drain |
| `fall_damage` | false | No fall damage |

### Common SafeZone Uses
- Server spawn areas
- Server hubs
- Trading centers / markets
- Event venues
- Tutorial areas
- NPC villages

<InfoBox variant="info">
SafeZones are typically used for server spawn points and communal areas where players should feel safe.
</InfoBox>

## WarZones

WarZones are designated combat areas where PvP is always enabled.

### WarZone Default Properties

| Flag | Default | Description |
|------|---------|-------------|
| `pvp_enabled` | true | PvP enabled |
| `friendly_fire` | true | Friendly fire enabled |
| `build_allowed` | false | No building |
| `container_access` | false | No container access |
| `interact_allowed` | true | Can use doors, buttons |
| `item_drop` | true | Can drop items |
| `item_pickup` | true | Can pick up items |
| `mob_spawning` | true | Mobs can spawn |
| `mob_damage` | true | Mobs can deal damage |
| `hunger_loss` | true | Hunger drains |
| `fall_damage` | true | Fall damage applies |

### Common WarZone Uses
- PvP arenas
- Event battlegrounds
- Resource competition areas
- King of the Hill zones

<InfoBox variant="tip">
WarZones are great for designated PvP areas where players can fight without affecting their faction's territorial power (when `warzonePowerLoss` is false).
</InfoBox>

## Zone Management

Only admins with `hyperfactions.admin.zones` permission can manage zones.

### Creating a Zone

1. **Create the named zone:**
   ```
   /f admin zone create <name> <safe|war>
   ```
   Example: `/f admin zone create spawn safe`

2. **Claim chunks for the zone:**
   Stand in each chunk and run:
   ```
   /f admin zone claim <name>
   ```

   Or use radius claiming:
   ```
   /f admin zone radius <name> <radius> [circle]
   ```

### Radius Claiming

Claim multiple chunks at once:

```
/f admin zone radius spawn 5
```
Claims a 5-chunk square (11x11 chunks total) centered on your position.

```
/f admin zone radius spawn 5 circle
```
Claims a circular area with 5-chunk radius.

### Managing Zones

**Remove a single chunk:**
```
/f admin zone unclaim
```

**Delete an entire zone:**
```
/f admin zone remove <name>
```
This removes the zone and unclaims all its chunks.

**List zones:**
```
/f admin zone list [safe|war]
```

<InfoBox variant="warning">
Zone changes take effect immediately. Players in the area will be subject to new rules instantly.
</InfoBox>

## Zone Flags

Each zone has 11 configurable flags that control behavior:

### Combat Flags
| Flag | Description |
|------|-------------|
| `pvp_enabled` | Allow player vs player damage |
| `friendly_fire` | Allow damage between faction/allied players |

### Building Flags
| Flag | Description |
|------|-------------|
| `build_allowed` | Allow block placement and breaking |
| `container_access` | Allow opening chests, furnaces, etc. |
| `interact_allowed` | Allow using doors, buttons, levers |

### Item Flags
| Flag | Description |
|------|-------------|
| `item_drop` | Allow dropping items |
| `item_pickup` | Allow picking up items |

### Environment Flags
| Flag | Description |
|------|-------------|
| `mob_spawning` | Allow mob spawning |
| `mob_damage` | Allow mobs to deal damage |
| `hunger_loss` | Allow hunger to drain |
| `fall_damage` | Allow fall damage |

### Modifying Zone Flags

Use the admin GUI to modify zone flags:

```
/f admin
```

Navigate to **Zones** → Select zone → **Configure Flags**

## Zone Configuration

### WarZone Power Loss
Control whether deaths in WarZones cause power loss:

```json
{
  "power": {
    "warzonePowerLoss": false
  }
}
```

| Value | Effect |
|-------|--------|
| `false` | No power loss in WarZones (default) |
| `true` | Normal power loss applies |

## Zone Priority

Zone rules take precedence over faction claims:

```
SafeZone > WarZone > Faction Claim > Wilderness
```

If a chunk is marked as a SafeZone:
- Previous faction claims are overridden
- SafeZone rules apply regardless of prior ownership

## Zone Storage

Zone data is stored in a single file:

```
plugins/HyperFactions/data/zones.json
```

This file contains all named zones with their chunks and flag configurations.

## Strategic Zone Placement

### Server Spawn
Create a SafeZone around spawn:
1. Create the zone: `/f admin zone create spawn safe`
2. Use radius: `/f admin zone radius spawn 3 circle`
3. Consider a buffer zone around the core

### PvP Arena
Create a WarZone for arenas:
1. Build your arena
2. Create zone: `/f admin zone create arena war`
3. Mark arena chunks: `/f admin zone radius arena 2`
4. Set `warzonePowerLoss: false` for penalty-free combat

### Resource Areas
Consider WarZones for contested resources:
- Encourages conflict over valuable areas
- No power penalties keep it fair
- Creates natural PvP hotspots

### Market/Trading Areas
Create a SafeZone for trading:
1. Create zone: `/f admin zone create market safe`
2. Claim the area
3. Players can trade safely without PvP risk
