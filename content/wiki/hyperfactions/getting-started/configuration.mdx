# Configuration

HyperFactions is highly configurable. This guide explains all available configuration options.

## Configuration File

The main configuration file is located at:

```
plugins/HyperFactions/config.json
```

<InfoBox variant="warning">
After making changes to the configuration, run `/f admin reload` to apply them without restarting the server.
</InfoBox>

## Configuration Sections

### Faction Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `maxMembers` | integer | 50 | Maximum members per faction |
| `maxNameLength` | integer | 24 | Maximum faction name length |
| `minNameLength` | integer | 3 | Minimum faction name length |
| `allowedNamePattern` | regex | `^[a-zA-Z0-9_]+$` | Allowed characters in faction names |

### Power Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `maxPlayerPower` | integer | 20 | Maximum power per player |
| `startingPower` | integer | 10 | Power new players start with |
| `powerPerClaim` | integer | 2 | Power required to maintain each claim |
| `regenPerMinute` | float | 0.0167 | Power regenerated per minute (~1/hour) |
| `deathPenalty` | integer | 1 | Power lost on death |
| `neutralKillPenalty` | integer | 1 | Power lost for killing neutral players |
| `warzonePowerLoss` | boolean | false | Whether to lose power on WarZone deaths |

<InfoBox variant="tip" title="Power Balance">
The `powerPerClaim` setting is crucial for balance. Higher values mean factions need more members to hold territory.
</InfoBox>

### Claims Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `maxClaims` | integer | 100 | Maximum claims per faction |
| `onlyAdjacent` | boolean | false | Require claims to be adjacent |
| `decayEnabled` | boolean | true | Enable inactive claim decay |
| `decayDelayDays` | integer | 7 | Days of inactivity before claims decay |
| `decayRatePerDay` | integer | 1 | Claims lost per day after decay starts |
| `showOnMap` | boolean | true | Show claims on the world map |
| `worldWhitelist` | array | `[]` | Only allow claims in these worlds (empty = all) |
| `worldBlacklist` | array | `[]` | Block claims in these worlds |

### Overclaim Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | boolean | true | Enable overclaiming system |
| `requireEnemyRelation` | boolean | true | Only allow overclaiming enemies |
| `powerThreshold` | integer | 0 | Enemy power threshold for overclaiming |

### Combat Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `tagDurationSeconds` | integer | 15 | Combat tag duration |
| `allyDamage` | boolean | false | Allow damage between allies |
| `factionDamage` | boolean | false | Allow damage between faction members |

### Teleport Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `warmupSeconds` | integer | 5 | Teleport warmup duration |
| `cooldownSeconds` | integer | 300 | Cooldown between teleports (in seconds) |
| `cancelOnMove` | boolean | true | Cancel teleport if player moves |
| `cancelOnDamage` | boolean | true | Cancel teleport if player takes damage |
| `crossWorld` | boolean | true | Allow cross-dimension teleportation |

### Stuck Command Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `warmupSeconds` | integer | 30 | Stuck command warmup duration |
| `cooldownSeconds` | integer | 300 | Cooldown between stuck uses (in seconds) |

### Invite Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `inviteExpirationMinutes` | integer | 5 | Minutes before invites expire |
| `joinRequestExpirationHours` | integer | 24 | Hours before join requests expire |

### Backup Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | boolean | true | Enable automatic backups |
| `hourlyRetention` | integer | 24 | Number of hourly backups to keep |
| `dailyRetention` | integer | 7 | Number of daily backups to keep |
| `weeklyRetention` | integer | 4 | Number of weekly backups to keep |
| `onShutdown` | boolean | true | Create backup on server shutdown |

<InfoBox variant="info" title="GFS Rotation">
Backups use a Grandfather-Father-Son rotation policy. Hourly backups are kept for 24 hours, then consolidated into daily, then weekly.
</InfoBox>

### Auto-Save Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | boolean | true | Enable automatic saves |
| `intervalMinutes` | integer | 5 | Minutes between auto-saves |

### Economy Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | boolean | false | Enable economy features |
| `currencyName` | string | "Coin" | Currency name (singular) |
| `currencyNamePlural` | string | "Coins" | Currency name (plural) |
| `currencySymbol` | string | "$" | Currency symbol |
| `startingBalance` | integer | 0 | Starting balance for new factions |

### Chat Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | boolean | true | Enable chat formatting |
| `format` | string | `[%faction%] %player%: %message%` | Chat format string |
| `tagDisplay` | string | "prefix" | Where to show faction tag (prefix/suffix/none) |
| `tagFormat` | string | `[%tag%]` | Format for the faction tag |
| `noFactionTag` | string | "" | Tag for players without a faction |
| `priority` | integer | 100 | Chat event priority |
| `relationColors.own` | hex | `#00FF00` | Color for own faction |
| `relationColors.ally` | hex | `#00FFFF` | Color for allies |
| `relationColors.enemy` | hex | `#FF0000` | Color for enemies |
| `relationColors.neutral` | hex | `#FFFF00` | Color for neutral factions |

### Territory Notifications

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | boolean | true | Show notifications when entering territories |

### Spawn Protection Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | boolean | true | Enable spawn protection |
| `durationSeconds` | integer | 10 | Duration of spawn protection |
| `breakOnAttack` | boolean | true | Remove protection if player attacks |
| `breakOnMove` | boolean | false | Remove protection if player moves |

### Faction Permissions Settings

Control the defaults and restrictions for faction territory permissions.

| Option | Type | Description |
|--------|------|-------------|
| `defaults` | object | Default values for each permission flag |
| `locks` | array | Flags that factions cannot change |
| `forced` | object | Flags forced to specific values |

**Available permission flags:**
- `pvp_enabled` - Allow PvP in territory
- `friendly_fire` - Allow faction members to damage each other
- `build_allowed` - Allow building in territory
- `container_access` - Allow container access
- `interact_allowed` - Allow interactions (doors, buttons, etc.)
- `item_drop` - Allow item dropping
- `item_pickup` - Allow item pickup
- `mob_spawning` - Allow mob spawning
- `mob_damage` - Allow mob damage
- `hunger_loss` - Allow hunger drain
- `fall_damage` - Allow fall damage

<InfoBox variant="tip" title="Locking Permissions">
Use `locks` to prevent factions from changing specific flags. Use `forced` to override values regardless of faction settings.
</InfoBox>

### Debug Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabledByDefault` | boolean | false | Start with debug mode enabled |
| `logToConsole` | boolean | true | Log debug messages to console |
| `categories` | array | `["claims", "power", "combat"]` | Active debug categories |

### Update Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | boolean | true | Enable update checking |
| `url` | string | GitHub API URL | Update check endpoint |
| `releaseChannel` | string | "stable" | Release channel (stable/beta) |

### Integration Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `hyperpermsEnabled` | boolean | true | Enable HyperPerms integration |
| `showFactionInChat` | boolean | true | Show faction name in chat |
| `showFactionOnMap` | boolean | true | Show faction territories on map |

### Message Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `prefix` | string | `[HyperFactions] ` | Message prefix |
| `primaryColor` | hex | `#00FFFF` | Primary text color |
| `secondaryColor` | hex | `#AAAAAA` | Secondary text color |
| `errorColor` | hex | `#FF5555` | Error message color |
| `successColor` | hex | `#55FF55` | Success message color |

## Common Configurations

### Competitive PvP Server

```json
{
  "power": {
    "maxPlayerPower": 10,
    "deathPenalty": 2,
    "regenPerMinute": 0.033
  },
  "combat": {
    "tagDurationSeconds": 30
  },
  "overclaim": {
    "enabled": true,
    "powerThreshold": 0
  },
  "factionPermissions": {
    "defaults": {
      "pvp_enabled": true
    }
  }
}
```

### Casual/Friendly Server

```json
{
  "power": {
    "maxPlayerPower": 50,
    "deathPenalty": 0
  },
  "combat": {
    "factionDamage": false,
    "allyDamage": false
  },
  "overclaim": {
    "enabled": false
  },
  "factionPermissions": {
    "defaults": {
      "pvp_enabled": false
    },
    "forced": {
      "pvp_enabled": false
    }
  }
}
```

### Roleplay Server

```json
{
  "claims": {
    "onlyAdjacent": true,
    "maxClaims": 200
  },
  "combat": {
    "tagDurationSeconds": 10
  },
  "teleport": {
    "warmupSeconds": 10
  },
  "chat": {
    "enabled": true,
    "tagDisplay": "prefix",
    "tagFormat": "[%tag%] "
  }
}
```
