## Admin Permissions

<PermissionTable
  title="Admin Permissions"

The `hyperfactions.admin` permission grants access to:
- `/f admin` menu
- `/f admin claim/unclaim`
- `/f admin bypass`
- `/f admin safezone add/remove`
- `/f admin warzone add/remove`
- `/f admin reload`

## Permission Defaults

| Default | Meaning |
|---------|---------|
| `true` | All players have this permission by default |
| `false` | Must be explicitly granted |
| `op` | Only server operators have this by default |

## HyperPerms Integration

When using HyperPerms, you can set these permissions per-group:

```yaml
groups:
  default:
    permissions:
      hyperfactions.use: true
      hyperfactions.create: true
      hyperfactions.home: true
  
  member:
    permissions:
      hyperfactions.*: true
      hyperfactions.admin: false
  
  admin:
    permissions:
      hyperfactions.*: true
```

## Permission by Role

While permissions control *who can attempt* an action, faction roles control *who can succeed*:

| Permission Has | Role Has | Result |
|----------------|----------|--------|
| ✓ Permission | ✓ Role | Success |
| ✓ Permission | ✗ Role | Role denied |
| ✗ Permission | ✓ Role | Permission denied |
| ✗ Permission | ✗ Role | Permission denied |

**Example:** A Member with `hyperfactions.promote` permission still cannot promote others because only Leaders can promote.

## Common Permission Setups

### Default Player
```
hyperfactions.use: true
hyperfactions.create: true
hyperfactions.claim: true
hyperfactions.home: true
hyperfactions.chat.faction: true
```

### VIP Player
```
hyperfactions.*: true
hyperfactions.admin: false
hyperfactions.bypass.warmup: true
```

### Moderator
```
hyperfactions.*: true
hyperfactions.admin: false
hyperfactions.bypass.protection: true
```

### Administrator
```
hyperfactions.*: true
```
